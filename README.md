# Beyond IID Data: A Proxy-Based Computational Study of Quantum Oracle Sketching Robustness under Structured Non-IID Streaming

[![DOI](https://zenodo.org/badge/DOI/10.5281/zenodo.19831893.svg)](https://doi.org/10.5281/zenodo.19831893)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)
[![Python](https://img.shields.io/badge/python-3.10%2B-blue.svg)](https://www.python.org/)
[![Reproducible](https://img.shields.io/badge/reproducible-yes-brightgreen.svg)](#reproducing-every-figure-and-table)

Reference implementation and reproducibility package for a proxy-based
computational sensitivity study of quantum oracle sketching robustness under
structured non-IID data streams.

> **Scope (read this first).** This is a *modelling study using heuristic
> proxies*, not a realised quantum-algorithmic benchmark. Quantum performance
> curves are paper-defined proxies derived from an effective-sample model;
> all classical baselines are empirical. No quantum circuit is simulated and
> no quantum algorithm is implemented. The central artefact is a
> $(\tau, r)$ diagnostic landscape, not a deployment recommendation.

### Synthetic
Five non-IID regimes are generated on demand from `code/data_generators.py`
with explicit seeds 0–9. No download required.

Two datasets from the [Numenta Anomaly Benchmark](https://github.com/numenta/NAB)
(NAB), MIT licence:

| File | Source | Size | Bins | Used for |
|---|---|---|---|---|
| `nyc_taxi.csv` | `numenta/NAB/data/realKnownCause/nyc_taxi.csv` | 265 KB | 10,320 (30-min) | Favourable-region case |

upstream:

```bash
  https://raw.githubusercontent.com/numenta/NAB/master/data/realKnownCause/nyc_taxi.csv

  https://raw.githubusercontent.com/numenta/NAB/master/data/realKnownCause/machine_temperature_system_failure.csv
```

```bash
# 1. Clone
git clone git@github.com:mmahmoudai/Beyond-IID-Data-A-Proxy-Based-Computational-Study-of-Quantum-Oracle-Sketchin.git
cd Beyond-IID-Data-A-Proxy-Based-Computational-Study-of-Quantum-Oracle-Sketchin

# 2. Set up environment
python -m venv venv
source venv/bin/activate    # Windows: venv\Scripts\activate
pip install -r code/requirements.txt

# 3. Reproduce every figure and table
python code/run_experiments.py     # Figures 1-7 and Table 3
python code/real_streams.py        # Figure 8 and Table 4 (real-stream sanity check)
python code/ablations.py           # Figure 9 and Table 6 (sensitivity ablations)
```

Total runtime: roughly 15–20 minutes on a standard laptop.

Outputs land in `figures/` (PDF + PNG) and `results/` (JSON summaries).

## Method in one paragraph

We define two operational correlation proxies — refreshing time $\tau$ and
repetition number $r$ — inspired by the correlated-data discussion in
[Zhao et al. (2026), arXiv:2604.07639](https://arxiv.org/abs/2604.07639). For
five structured non-IID regimes (IID, Markov switching, seasonal drift, burst
repetition, long-range dependence), we (i) specify analytic proxy expressions
for $\tau$ and $r$, (ii) measure their empirical counterparts on generated
streams, (iii) compare three classical memory-bounded streaming baselines
against a heuristic quantum accuracy proxy
$\mathrm{acc\_proxy} = \max(0.5,\, 1 - C^2 n \log(1/\delta) / T_{\text{eff}})$
with $T_{\text{eff}} = T/(\tau r)$, $C = 2$, $\delta = 0.05$, and (iv) map
the regimes onto a $(\tau, r)$ sensitivity landscape that we treat as a
hypothesis-generating diagnostic, not deployment guidance. The framework is
applied end-to-end to two real NAB streams (Section 6.8) as a sanity check;
NYC Taxi falls in the proxy-favourable region while Machine Temperature
falls in the unfavourable saturation region. Four ablations (Section 6.9)
verify the qualitative landscape ordering is robust to alternative $\tau$
estimators, forward-window size, stream length, and target-function choice.

## What this study does not do

- It does **not** simulate quantum circuits.
- It does **not** implement a quantum oracle sketching algorithm.
- It does **not** prove that the paper-defined $\tau$ and $r$ are formally
  equivalent to the exact correlation parameters in the Zhao et al. framework.
- It does **not** establish a domain-wide validation across financial,
  network-traffic, or additional sensor streams — only two NAB sanity-check
  cases are examined.
- The classical baselines are practical learners, not the worst-case
  task-specific instances that motivate the $\Omega(\sqrt{N})$ classical
  hardness reference.

These deferments are explicit follow-on targets in Section 7.4 of the
manuscript.
